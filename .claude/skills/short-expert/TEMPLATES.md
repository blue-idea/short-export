# 统一模板格式定义

本文件定义了短剧创作过程中所有输出的标准 Markdown 格式。**所有输出必须严格遵循这些模板**。

---

---

## 一、项目信息模板

**文件名**：`00_项目信息.md`

```markdown
# 《项目标题》

## 基本信息

| 字段 | 内容 |
|---|---|
| **标题** | 《XXX》 |
| **类型** | 甜宠/复仇/逆袭/穿越/战神/赘婿 |
| **集数** | XX集 |
| **单集时长** | 1-2分钟 |
| **目标受众** | 女性向/男性向/全年龄 |
| **创建日期** | YYYY-MM-DD |

## 一句话卖点

> [身份反差] + [核心困境] + [关键行动] + [悬念钩子]

示例：
> 被渣男当众退婚的她，转身嫁给了全城最神秘的男人。所有人等着看笑话，却不知她的新婚丈夫，是让渣男跪着叫爸爸的存在……

## 核心爽点

- **主爽点**：打脸/逆袭/身份反转/宠溺/碾压...
- **副爽点**：...

## 标题验证

- [ ] 有"网感"（反差/悬念/身份标签/数字）
- [ ] 能引发好奇心
- [ ] 暗示核心冲突
- [ ] 目标受众会点击

## 备注

（其他需要记录的信息）
```

> ⚠️ **这是故事线的核心，重在剧情的连贯性、合理性和内容的饱满度**  
> ⚠️ **无需关注具体场景分镜、情绪卡点和付费设计**  
> ⚠️ **每集字数要求：300-500字**

### 第1集：[集名/核心事件]

**剧情详情**：
[这里撰写完整的故事内容。侧重描述发生了什么，人物做了什么选择，结果如何。]

[包含：起因 -> 经过 -> 结果 -> 下集引子]

**逻辑与关键信息**（简要标注）：
- 【关键转折】：[本集剧情的关键转折点]
