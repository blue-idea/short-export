# 故事线编写与一致性保障

## 核心理念

> **故事线是剧本的"逻辑蓝图"，一致性保障应"预防优于修复"**

**故事线定位**：
- 战略层规划，聚焦因果逻辑和关键转折
- 是生成大纲和剧本的核心依据
- 不包含场景细节、台词方向、情绪设计

**一致性保障**：
- 在故事线阶段就嵌入检查机制
- 每10集立即自检，发现问题立即调整
- 避免问题累积到剧本阶段才修复

---

## 第一部分：故事线编写方法

### 1. 故事线文档结构

故事线文档（`03_完整故事线/`）应包含以下核心部分：

```
03_完整故事线/
├── 第一阶段故事线.md（1-30集）
├── 第二阶段故事线.md（31-60集）
└── ...
```

每个文件包含：
- ⭐逐集剧情逻辑（核心必填，150-200字/集）
- 辅助管理表格（可选，如时间线、状态表）

### 2. 逐集剧情逻辑撰写

#### 2.1 核心内容框架（150-200字）

每集必须包含以下内容：

**核心事件**（一句话，20-30字）：
> 本集发生的主要事件

**因果链条**（100-150字）：
- **起因**：为什么会发生这件事？基于前集什么情节？
- **经过**：事件如何发展？人物做了什么选择？为什么这么选？
- **结果**：导致什么后果？引发下集什么事件？

**关键转折**（一句话）：
> 本集剧情走向的改变点

**人物状态标注**：
- 【人物状态】主角/重要角色的状态变化（能力/认知/情感）

**伏笔标注**（如有）：
- 【伏笔F00X】伏笔内容
- 【回收F00X】回收内容

#### 2.2 撰写要点

✅ **必须包含**：
- 完整的因果逻辑链条
- 人物动机的合理性
- 关键转折点的标注
- 人物状态的变化追踪

❌ **不要包含**：
- 场景拆解（留给大纲）
- 台词方向（留给剧本）
- 情绪曲线细节（留给大纲）
- 卡点具体设计（留给大纲）
- 付费点情绪设计（留给付费点分析）

#### 2.3 质量标准

**逻辑性检查**：
- [ ] 因果关系清晰：每个事件都有明确的前因后果
- [ ] 人物动机合理：角色行为符合其性格和状态
- [ ] 时间线连续：事件发生顺序合理，无突兀跳跃
- [ ] 空间逻辑：人物位置转换符合实际

**连贯性检查**：
- [ ] 前后集衔接自然：本集起始承接上集结尾
- [ ] 状态延续：人物状态变化有迹可循
- [ ] 信息传递合理：角色知道的信息有来源依据

**饱满度检查**：
- [ ] 故事内容充实：非流水账，有冲突和转折
- [ ] 事件完整：有起因、经过、结果的闭环

### 3. ⭐逻辑自检机制（每10集必做）

> **第一道检查防线**：在故事线阶段就发现并修复问题

#### 3.1 触发时机

- 每完成10集故事线
- 发现逻辑问题时
- 人物状态重大变化时

#### 3.2 快速检查清单（3分钟）

**时间线检查**：
- [ ] 这10集的时间是否连续？
- [ ] 有无"突然到了明天"却没交代？
- [ ] 时间跨度是否合理？

**空间逻辑检查**：
- [ ] 人物位置转换是否合理？
- [ ] 有无"瞬移"现象？
- [ ] 场景转换是否有交代？

**能力一致性检查**：
- [ ] 角色能力有无突然变化？
- [ ] 是否存在"强行降智"？
- [ ] 能力提升是否有合理解释？

**伏笔记录检查**：
- [ ] 新埋的伏笔是否在备忘录中记录？
- [ ] 伏笔ID是否唯一？
- [ ] 计划回收位置是否明确？

**人设一致性检查**：
- [ ] 角色行为是否符合性格设定？
- [ ] 有无"人设崩塌"现象？
- [ ] 性格变化是否有铺垫？

#### 3.3 发现问题的处理原则

1. **立即在故事线中调整**，不要继续往下写
2. **不要积累问题**到大纲或剧本阶段
3. **同步更新**人物状态追踪表
4. **记录修改**，方便后续回溯

## 第二部分：一致性保障机制

### 1. 三道检查防线总览

| 防线 | 阶段 | 时机 | 耗时 | 重点 |
|---|---|---|---|---|
| 第一道 | 故事线 | 每10集 | 3分钟 | 逻辑一致性 |
| 第二道 | 大纲 | 每集 | 2分钟 | 引用准确性 |
| 第三道 | 剧本 | 每10集 | 5分钟 | 表达质量 |

### 2. 第一道防线：故事线阶段

**详见上述"逻辑自检机制"**

### 3. 第二道防线：大纲阶段

#### 3.1 强制检查清单（2分钟）

**引用准确性**：
- [ ] 核心事件是否与故事线完全一致？
- [ ] 是否一字不差地复制了因果链条？

**无新增剧情**：
- [ ] 是否添加了故事线中没有的情节？
- [ ] 场景拆解是否基于故事线内容？

**人物状态一致性**：
- [ ] 人物状态是否与故事线追踪表一致？
- [ ] 能力/认知/情感是否有突变？

**承接故事线**：
- [ ] 场景拆解是否服务于故事线的因果链条？
- [ ] 卡点是否基于故事线的关键转折？

#### 3.2 处理原则

- **发现不一致立即回顾故事线**
- **禁止凭想象添加内容**
- **必须引用原文，不能改写**

### 4. 第三道防线：剧本阶段

#### 4.1 轻量级质量检查（每10集，5分钟）

> ✅ 前两道防线已保障逻辑一致性，本阶段聚焦表达质量

**遵循大纲**：
- [ ] 是否严格按照大纲的场景拆解写作？
- [ ] 有无脱离大纲自由发挥？

**台词口语化**：
- [ ] 是否删掉了所有连接词（因为/所以/但是）？
- [ ] 对话是否简短、口语化？

**动作外化**：
- [ ] 是否用△动作替代心理描写？
- [ ] 是否避免了"不可拍"的内容？

**卡点毒性**：
- [ ] 本集卡点毒性是否达到★★★以上？
- [ ] 切断时机是否精准？

**情绪波动**：
- [ ] 每30秒是否有情绪波动点？
- [ ] 情绪起伏是否合理？

**悬念接力**：
- [ ] 是否做到解决30%旧悬念，制造100%新悬念？
- [ ] 三层悬念是否都在推进？

#### 4.2 不需要检查的内容

- ❌ 逻辑一致性（第一道防线已保障）
- ❌ 人物状态一致性（第一道防线已保障）
- ❌ 剧情冲突（第二道防线已检查）

### 5. 常见一致性问题及解决方案

#### 问题1：角色"能力失忆"

**表现**：
- 第5集能轻松打败10个人
- 第15集却被3个人围攻
- 中间没有任何受伤或能力下降的交代

**解决方案**：
1. 在故事线第15集增加解释（刚做完手术、被下药）
2. 或修改第5集，降低战斗能力描述
3. 在故事线中明确标注角色的能力基准线

#### 问题2：信息传递不合理

**表现**：
- 角色A在第10集说出了只有角色B知道的秘密
- 但前面从未交代过B告诉A

**解决方案**：
1. 在故事线第8-9集补充B告诉A的情节
2. 或在第10集标注"A偷听到的"
3. 或修改第10集，让A的内容不涉及这个秘密

#### 问题3：时间线混乱

**表现**：
- 第12集说"三天后的婚礼"
- 第13集直接是婚礼
- 中间只隔了1集（约2分钟）

**解决方案**：
1. 在故事线第13集标注时间跳跃："三天后"
2. 或修改第12集为"明天的婚礼"
3. 在故事线中明确标注时间跨度

#### 问题4：伏笔回收矛盾

**表现**：
- 第3集埋伏笔：项链是母亲留下的唯一遗物
- 第25集回收：原来项链有两条，一条在姐姐那里

**解决方案**：
1. 修改故事线第3集：项链是母亲留下的重要遗物（去掉"唯一"）
2. 或修改第25集：姐姐的是复制品/假货
3. 在伏笔表中记录详细信息，保证前后一致

#### 问题5：情绪断崖

**表现**：
- 第18集：女主对男主恨之入骨
- 第19集：女主开始心动
- 中间没有任何转变契机

**解决方案**：
1. 在故事线第18集结尾增加"男主救女主"情节
2. 或在第19集标注情感OS："虽然还恨他，但那天他的眼神..."
3. 或将"心动"延后到第22-23集，中间增加情感过渡

## 核心原则

> ⚠️ **预防大于修正**：在故事线阶段就严格遵循逻辑

> ⚠️ **及时检查**：每10集必须停下来检查，不积累问题

> ⚠️ **文档同步**：所有修改必须同步到故事线，否则后续问题更多

> ⚠️ **三道防线各司其职**：
> - 故事线保障逻辑
> - 大纲保障引用准确
> - 剧本保障表达质量

**记住**：一致性不是限制创作，而是保证观众体验。只有逻辑自洽的剧本，才能让观众全身心投入，从而提高付费转化率。
